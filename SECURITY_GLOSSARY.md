# Cybersecurity Glossary

A comprehensive glossary of cybersecurity terms, acronyms, and concepts. Curated by [AllSecurityNews.com](https://allsecuritynews.com)

## A

**Access Control** - Security mechanisms that regulate who or what can view or use resources in a computing environment.

**Advanced Persistent Threat (APT)** - A prolonged and targeted cyberattack in which an intruder gains access to a network and remains undetected for an extended period.

**Adware** - Software that automatically displays or downloads advertising material when a user is online.

**Air Gap** - A security measure that involves physically isolating a computer or network from unsecured networks, including the internet.

**Antivirus** - Software designed to detect, prevent, and remove malware from computer systems.

**API Security** - Practices and technologies used to protect Application Programming Interfaces from attacks and abuse.

**Attack Surface** - The total sum of vulnerabilities, entry points, and assets that an attacker could exploit to gain unauthorized access.

**Attack Vector** - The method or pathway used by an attacker to access a victim's system or network.

**Authentication** - The process of verifying the identity of a user, device, or system before granting access to resources.

**Authorization** - The process of determining what resources or actions an authenticated user is allowed to access or perform.

## B

**Backdoor** - A hidden method of bypassing normal authentication or encryption to gain unauthorized access to a system.

**Baseline** - A security configuration standard used as a reference point for system hardening and compliance.

**BEC (Business Email Compromise)** - A sophisticated scam targeting businesses that regularly perform wire transfers and have suppliers abroad.

**Biometrics** - Authentication methods based on unique physical characteristics such as fingerprints, facial recognition, or iris scans.

**Blacklist** - A list of entities (IP addresses, applications, users) that are denied access or blocked from a system.

**Blue Team** - The defensive security team responsible for protecting systems and detecting/responding to threats.

**Botnet** - A network of compromised computers (bots) controlled remotely by an attacker to perform malicious activities.

**Breach** - An incident where data, computer systems, or networks are accessed or exposed without authorization.

**Brute Force Attack** - An attack method that tries every possible combination of passwords or encryption keys until the correct one is found.

**Bug Bounty** - A program offered by organizations that rewards individuals for discovering and reporting security vulnerabilities.

## C

**C2 (Command and Control)** - The infrastructure used by attackers to communicate with compromised systems in a botnet.

**Certificate Authority (CA)** - A trusted entity that issues digital certificates for verifying identities on the internet.

**CISO (Chief Information Security Officer)** - The executive responsible for an organization's information and data security strategy.

**Cloud Security** - The collection of policies, technologies, and controls deployed to protect data, applications, and infrastructure in cloud computing environments.

**Compliance** - Adherence to laws, regulations, and standards related to information security (e.g., GDPR, HIPAA, PCI DSS).

**Credential Stuffing** - An attack where stolen usernames and passwords from one service are used to attempt access to other services.

**Cryptojacking** - Unauthorized use of someone's computing resources to mine cryptocurrency.

**Cryptography** - The practice of securing information by transforming it into an unreadable format that can only be decrypted with the proper key.

**CSRF (Cross-Site Request Forgery)** - An attack that tricks a victim into submitting a malicious request using their authenticated session.

**CVE (Common Vulnerabilities and Exposures)** - A standardized identifier for known security vulnerabilities maintained by MITRE Corporation.

**CVSS (Common Vulnerability Scoring System)** - A framework for rating the severity of security vulnerabilities on a scale from 0-10.

**CWE (Common Weakness Enumeration)** - A community-developed list of software and hardware weakness types.

**Cyber Kill Chain** - A framework for understanding the stages of a cyberattack, from reconnaissance to data exfiltration.

## D

**Dark Web** - The encrypted, hidden part of the internet often used for anonymous communication and illegal activities.

**Data Breach** - An incident where sensitive, protected, or confidential data is accessed or disclosed without authorization.

**Data Loss Prevention (DLP)** - Technologies and processes designed to ensure sensitive data is not lost, misused, or accessed by unauthorized users.

**DDoS (Distributed Denial of Service)** - An attack that overwhelms a target system with traffic from multiple sources, making it unavailable to legitimate users.

**Decompiler** - A tool that converts executable code back into source code for analysis.

**Defense in Depth** - A security strategy that employs multiple layers of defense to protect against threats.

**DevSecOps** - The practice of integrating security into every phase of the software development lifecycle.

**DMZ (Demilitarized Zone)** - A physical or logical subnet that separates an internal network from untrusted external networks.

**DNS Poisoning** - An attack that corrupts DNS cache data to redirect users to malicious websites.

**Domain Hijacking** - The act of changing a domain's registration without the permission of the original owner.

**Doxing** - The act of publicly revealing previously private personal information about an individual or organization.

## E

**EDR (Endpoint Detection and Response)** - Security tools that monitor end-user devices to detect and respond to cyber threats.

**Encryption** - The process of encoding information so that only authorized parties can access it.

**Endpoint Security** - Security measures designed to protect end-user devices (computers, mobile devices) from cyber threats.

**Exploit** - Code or technique that takes advantage of a vulnerability to compromise a system.

**Exploit Kit** - A toolkit used by attackers to probe systems for vulnerabilities and deliver malicious payloads.

## F

**False Positive** - When a security tool incorrectly identifies legitimate activity as malicious.

**Fileless Malware** - Malicious software that operates in memory without writing files to disk, making it harder to detect.

**Firewall** - A network security system that monitors and controls incoming and outgoing network traffic based on predetermined security rules.

**Forensics (Digital Forensics)** - The process of collecting, analyzing, and preserving electronic evidence for investigation purposes.

**Fuzzing** - A software testing technique that involves providing invalid, unexpected, or random data to a program to find vulnerabilities.

## G

**GDPR (General Data Protection Regulation)** - European Union regulation on data protection and privacy.

**Grayware** - Software that falls between legitimate programs and malware, often including adware and spyware.

**Grey Hat Hacker** - A hacker who operates with ambiguous intent, sometimes violating laws but not for personal gain.

## H

**Hashing** - A one-way cryptographic function that converts data into a fixed-size string of characters.

**HIPAA (Health Insurance Portability and Accountability Act)** - U.S. legislation that provides data privacy and security provisions for safeguarding medical information.

**Honeypot** - A decoy system designed to attract attackers and study their methods without risking real assets.

**HTTPS (Hypertext Transfer Protocol Secure)** - The secure version of HTTP that encrypts data in transit between browser and server.

## I

**IAM (Identity and Access Management)** - Framework of policies and technologies for ensuring proper access to resources.

**IDS (Intrusion Detection System)** - A system that monitors network traffic for suspicious activity and alerts administrators.

**Incident Response** - The process of identifying, investigating, and remediating security incidents.

**Indicators of Compromise (IoC)** - Forensic artifacts that suggest a system has been breached.

**Information Security** - The practice of protecting information by mitigating information risks.

**IPS (Intrusion Prevention System)** - A system that monitors network traffic and can automatically block detected threats.

**ISO 27001** - International standard for information security management systems.

## J

**Jailbreaking** - The process of removing software restrictions on iOS devices to gain root access.

**JavaScript Injection** - An attack that inserts malicious JavaScript code into web applications.

## K

**Kerberos** - A network authentication protocol that uses tickets to allow secure authentication over non-secure networks.

**Keylogger** - Malicious software or hardware that records every keystroke made on a device.

**Kill Chain** - See Cyber Kill Chain.

## L

**Lateral Movement** - Techniques used by attackers to progressively move through a network after initial compromise.

**LDAP (Lightweight Directory Access Protocol)** - A protocol for accessing and maintaining distributed directory information services.

**Least Privilege** - Security principle that users should only have the minimum access rights needed to perform their duties.

**Log Analysis** - The process of reviewing and interpreting security logs to identify threats and incidents.

## M

**Malvertising** - The use of online advertising to spread malware.

**Malware** - Malicious software designed to harm, exploit, or otherwise compromise computer systems.

**Man-in-the-Middle (MitM)** - An attack where an attacker secretly intercepts and relays communications between two parties.

**MDM (Mobile Device Management)** - Software that secures, monitors, and manages mobile devices deployed across an organization.

**Metasploit** - A popular penetration testing framework used by security professionals and attackers.

**MITRE ATT&CK** - A knowledge base of adversary tactics and techniques based on real-world observations.

**Multi-Factor Authentication (MFA)** - An authentication method that requires two or more verification factors to gain access.

## N

**NAC (Network Access Control)** - Security approach that restricts unauthorized users and devices from accessing a network.

**NGFW (Next-Generation Firewall)** - Advanced firewall that includes features like application awareness and intrusion prevention.

**NIST (National Institute of Standards and Technology)** - U.S. agency that develops cybersecurity standards and frameworks.

**NVD (National Vulnerability Database)** - U.S. government repository of standards-based vulnerability management data.

## O

**OAuth** - An open standard for access delegation commonly used for token-based authentication.

**OSINT (Open Source Intelligence)** - Intelligence collected from publicly available sources.

**OWASP (Open Web Application Security Project)** - Non-profit organization focused on improving software security.

**OWASP Top 10** - Annual list of the most critical web application security risks.

## P

**Patch** - A software update designed to fix vulnerabilities or improve functionality.

**Payload** - The part of malware that performs the malicious action after a system is compromised.

**PCI DSS (Payment Card Industry Data Security Standard)** - Security standards for organizations that handle credit card information.

**Pen Testing (Penetration Testing)** - Authorized simulated attacks on computer systems to find security vulnerabilities.

**Pharming** - An attack that redirects website traffic to fraudulent sites without the user's knowledge.

**Phishing** - A social engineering attack that uses deceptive emails or messages to trick victims into revealing sensitive information.

**PKI (Public Key Infrastructure)** - Framework for creating, managing, and revoking digital certificates.

**Polymorphic Malware** - Malware that constantly changes its code to evade detection.

**Port Scanning** - The process of probing a system for open network ports to identify services and potential vulnerabilities.

**Privilege Escalation** - An attack that exploits vulnerabilities to gain elevated access rights on a system.

**Purple Team** - A collaborative security team combining red and blue team approaches for comprehensive testing.

## Q

**Quarantine** - The isolation of suspected malware or infected files to prevent further damage.

## R

**Ransomware** - Malware that encrypts victim's files and demands payment for the decryption key.

**RAT (Remote Access Trojan)** - Malware that allows attackers to remotely control an infected computer.

**Red Team** - The offensive security team that simulates attacks to test an organization's defenses.

**Reverse Engineering** - The process of analyzing software or hardware to understand its components and functionality.

**Risk Assessment** - The process of identifying, analyzing, and evaluating security risks to an organization.

**Rootkit** - Malware designed to hide the existence of certain processes or programs from normal detection methods.

## S

**Sandboxing** - A security mechanism for separating running programs to contain threats and prevent system-wide damage.

**SAST (Static Application Security Testing)** - Analysis of application source code to find security vulnerabilities.

**Scareware** - Malware that frightens users into taking action, often by displaying fake security alerts.

**Script Kiddie** - An inexperienced hacker who uses existing tools and scripts created by others.

**Security Posture** - The overall cybersecurity strength and resilience of an organization.

**Session Hijacking** - An attack where an attacker takes over a valid user session.

**SIEM (Security Information and Event Management)** - A platform that collects and analyzes security event data from across an organization.

**Social Engineering** - Manipulation techniques used to trick people into divulging confidential information or performing actions that compromise security.

**SOC (Security Operations Center)** - A centralized unit that monitors, detects, and responds to cybersecurity incidents.

**Spear Phishing** - A targeted phishing attack directed at specific individuals or organizations.

**Spoofing** - The act of disguising communication from an unknown source as being from a known, trusted source.

**Spyware** - Software that secretly monitors and collects user information without their knowledge.

**SQL Injection** - An attack that inserts malicious SQL code into application queries to manipulate databases.

**SSL/TLS** - Cryptographic protocols that provide secure communication over networks.

**Supply Chain Attack** - An attack that targets less-secure elements in the supply chain to compromise the final target.

## T

**Threat Actor** - An individual or group responsible for security incidents or attacks.

**Threat Hunting** - The proactive search for cyber threats that have evaded existing security solutions.

**Threat Intelligence** - Evidence-based knowledge about existing or emerging threats used to inform security decisions.

**Threat Modeling** - A process for identifying, assessing, and prioritizing potential threats to a system.

**Tokenization** - The process of replacing sensitive data with non-sensitive substitutes (tokens).

**Trojan Horse** - Malware disguised as legitimate software that performs malicious actions once installed.

**Two-Factor Authentication (2FA)** - A security process requiring two different authentication factors (e.g., password + SMS code).

## U

**UAC (User Account Control)** - A Windows security feature that helps prevent unauthorized changes to the operating system.

**URL Filtering** - Technology that blocks access to certain websites based on their URLs.

## V

**Vishing** - Voice phishing - using phone calls to trick victims into revealing sensitive information.

**VLAN (Virtual Local Area Network)** - Logical grouping of network devices for improved security and performance.

**VPN (Virtual Private Network)** - A secure, encrypted connection over a public network.

**Vulnerability** - A weakness in a system that could be exploited to compromise security.

**Vulnerability Assessment** - The process of identifying, quantifying, and prioritizing vulnerabilities in a system.

**Vulnerability Scanner** - Automated tool that scans systems for known security weaknesses.

## W

**WAF (Web Application Firewall)** - A firewall specifically designed to protect web applications from attacks.

**Watering Hole Attack** - An attack that targets a specific group by infecting websites they commonly visit.

**Whaling** - A type of spear phishing attack that targets high-profile executives or important individuals.

**Whitelisting** - A security practice of explicitly allowing only approved applications, users, or IP addresses.

**Worm** - Self-replicating malware that spreads across networks without human interaction.

## X

**XDR (Extended Detection and Response)** - Security solution that provides integrated threat detection and response across multiple security layers.

**XSS (Cross-Site Scripting)** - An attack that injects malicious scripts into web pages viewed by other users.

## Y

**YARA** - A tool used to identify and classify malware samples based on textual or binary patterns.

## Z

**Zero-Day** - A vulnerability that is exploited before the vendor releases a patch.

**Zero Trust** - A security model that requires strict identity verification for every person and device trying to access resources.

**Zombie** - A computer that has been compromised and is being controlled remotely as part of a botnet.
